# 飞扬俱乐部电脑维修管理系统 - 用户使用指南

## 系统简介

飞扬俱乐部电脑维修管理系统是专为四川大学飞扬俱乐部"大修"活动设计的电子化管理系统，支持完整的维修业务流程，从订单录入到维修完成的全程管理。

## 角色分工

### 1号位 & 4号位 - 订单录入员
- **职责**：负责录入新的维修订单
- **分工**：1号位录入奇数编号(0001, 0003...)，4号位录入偶数编号(0002, 0004...)
- **页面**：订单录入页面

### 5号位 - 客服管理员
- **职责**：发送维修完成短信通知，处理订单完成
- **页面**：5号位大屏、订单管理页面

### 6号位 - 技术员调度员
- **职责**：分配技术员，管理维修进度
- **页面**：6号位大屏、技术员管理页面

## 快速开始

### 首次使用

1. **访问系统**：打开浏览器访问系统地址
2. **初始化数据库**：首次使用需访问 `/api/init_database.php` 初始化数据库
3. **选择角色**：根据工作岗位选择对应的功能页面

### 主要功能导航

- **控制台**：系统概览和统计数据
- **订单录入**：1、4号位录入新订单
- **订单管理**：查看和管理所有订单
- **技术员管理**：管理技术员信息和工作分配
- **6号位大屏**：技术员调度专用界面
- **5号位大屏**：客服管理专用界面

## 详细操作指南

### 📝 订单录入操作（1、4号位）

#### 基本录入流程

1. **选择录入位置**
   - 1号位：选择"1号位"，系统自动生成奇数编号
   - 4号位：选择"4号位"，系统自动生成偶数编号

2. **填写客户信息**
   ```
   客户姓名：必填，客户真实姓名
   联系电话：必填，11位手机号码
   QQ号码：选填，用于联系客户
   ```

3. **填写设备信息**
   ```
   设备类型：必填，如"笔记本电脑"、"台式机"等
   设备型号：选填，具体型号信息
   故障描述：必填，详细描述故障现象
   配件情况：选填，记录配件完整性
   备注信息：选填，其他需要说明的信息
   ```

4. **提交订单**
   - 点击"提交订单"：提交当前订单
   - 点击"提交并添加下一个"：提交后自动清空表单，继续录入下一个订单

#### 快捷操作

- **快捷键**：`Ctrl + Enter` 快速提交订单
- **表单重置**：点击"重置表单"清空所有内容
- **自动编号**：切换位置时自动更新订单编号

#### 注意事项

- 订单编号由系统自动生成，无需手动输入
- 手机号格式：必须是11位数字，以1开头
- 故障描述建议详细填写，便于技术员了解问题
- 提交成功后订单状态自动设为"待接单"

### 🔄 工作流管理（订单状态流转）

#### 订单状态说明

```
待接单 → 维修中 → 待取机 → 已完成
```

- **待接单**：订单已录入，等待技术员接单
- **维修中**：技术员已接单，正在维修
- **待取机**：维修完成，等待客户取机
- **已完成**：客户已取机，订单完成

#### 状态操作权限

- **6号位**：待接单 → 维修中（分配技术员）
- **技术员**：维修中 → 待取机（完成维修）
- **5号位**：待取机 → 已完成（客户取机）

#### 状态变更操作

1. **在订单管理页面**
   - 找到需要操作的订单
   - 点击"状态"列的下拉菜单
   - 选择目标状态
   - 确认操作

2. **在大屏界面**
   - 直接点击订单行的操作按钮
   - 选择相应操作（接单、完成等）

### 👨‍💻 技术员管理

#### 技术员信息管理

1. **添加技术员**
   - 进入"技术员管理"页面
   - 点击"添加技术员"按钮
   - 填写基本信息：姓名、电话、专业领域
   - 设置初始状态：在线/离线

2. **编辑技术员信息**
   - 在技术员列表中点击"编辑"按钮
   - 修改相关信息
   - 保存更改

3. **状态管理**
   - **在线**：可以接收新订单
   - **忙碌**：正在处理订单，暂不接新单
   - **离线**：不在工作状态

#### 技术员分配流程

1. **6号位操作**
   - 在6号位大屏查看待接单订单
   - 点击订单的"分配技术员"按钮
   - 选择合适的技术员
   - 确认分配

2. **转单操作**
   - 当前技术员无法处理时
   - 在6号位大屏点击"转单"
   - 选择新的技术员
   - 填写转单原因
   - 确认转单

### 📱 短信通知功能

#### 发送单条短信

1. **在订单详情页面**
   - 点击"发送短信"按钮
   - 选择短信模板或自定义内容
   - 预览短信内容
   - 确认发送

2. **常用模板**
   - **维修完成通知**：设备维修完成，请取机
   - **维修开始通知**：设备已开始维修
   - **维修延期通知**：维修需要延期
   - **自定义消息**：自由编写短信内容

#### 批量发送短信

1. **在5号位大屏**
   - 查看待取机订单列表
   - 选择需要通知的订单
   - 点击"批量发送短信"
   - 选择模板
   - 确认批量发送

2. **发送进度监控**
   - 实时显示发送进度
   - 统计成功/失败数量
   - 查看发送结果

#### 短信历史管理

- **查看历史**：点击"短信历史"查看所有发送记录
- **导出记录**：将发送历史导出为CSV文件
- **消息详情**：查看具体的短信内容

### 📺 大屏显示功能

#### 6号位大屏 - 技术员调度

**主要功能**：
- 实时显示待接单和维修中的订单
- 技术员在线状态监控
- 订单分配和转单操作

**操作流程**：
1. 查看待接单订单列表
2. 点击"分配技术员"按钮
3. 在弹窗中选择技术员
4. 确认分配，订单状态变为"维修中"

**快捷键**：
- `F5`：手动刷新数据
- `空格`：暂停/恢复自动刷新

#### 5号位大屏 - 客服管理

**主要功能**：
- 显示待取机订单列表
- 发送维修完成短信通知
- 订单完成操作

**操作流程**：
1. 查看待取机订单
2. 点击"发送短信"通知客户
3. 客户来取机时点击"完成订单"
4. 订单状态变为"已完成"

### 📊 数据统计和报表

#### 控制台统计

- **今日数据**：今日新增、完成订单数量
- **状态分布**：各状态订单数量统计
- **技术员状态**：在线技术员数量
- **处理效率**：平均处理时间等

#### 订单报表

- **按状态筛选**：查看特定状态的订单
- **按时间筛选**：查看特定时间段的订单
- **按技术员筛选**：查看特定技术员的订单

## 常见问题解答

### Q1：订单编号重复怎么办？
**A**：系统会自动检测编号重复，如遇重复会自动跳到下一个可用编号。

### Q2：技术员不在线时如何分配订单？
**A**：可以分配给离线技术员，但建议优先分配给在线技术员以提高处理效率。

### Q3：短信发送失败怎么处理？
**A**：检查手机号格式是否正确，可以在短信历史中查看失败原因，必要时可重新发送。

### Q4：如何查看订单的完整处理历史？
**A**：在订单管理页面点击订单的"查看"按钮，可以看到完整的操作日志。

### Q5：大屏显示数据不更新怎么办？
**A**：按F5手动刷新，或检查网络连接是否正常。

### Q6：忘记保存订单信息怎么办？
**A**：系统支持草稿保存功能，刷新页面后可以恢复之前填写的内容。

## 最佳实践建议

### 录入规范

1. **客户信息**
   - 姓名使用真实姓名，避免昵称
   - 电话号码仔细核对，确保准确
   - 故障描述尽量详细，包含具体现象

2. **设备信息**
   - 设备类型标准化，如"笔记本电脑"、"台式机主机"
   - 型号信息尽量完整，便于配件采购
   - 配件情况详细记录，避免纠纷

### 工作流程

1. **高效分配**
   - 根据技术员专长分配订单
   - 平衡各技术员工作量
   - 优先处理紧急订单

2. **及时沟通**
   - 维修开始时发送通知短信
   - 维修完成后立即通知客户
   - 遇到问题及时与客户沟通

3. **状态管理**
   - 及时更新订单状态
   - 记录详细的维修备注
   - 完成后及时归档

### 数据管理

1. **定期备份**
   - 建议每日备份重要数据
   - 保留历史记录便于查询
   - 定期清理过期数据

2. **统计分析**
   - 定期查看处理效率统计
   - 分析常见故障类型
   - 优化工作流程
